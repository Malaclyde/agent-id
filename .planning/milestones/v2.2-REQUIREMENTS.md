## v2.2 Requirements (Final Status)

### Agent Demo - Configuration (ACONF)

- [x] **ACONF-01**: User can configure agent demo via .env file with backend URL and Ed25519 keys. — **Validated**
- [x] **ACONF-02**: User can generate Ed25519 keypairs and save to .env file. — **Validated**
- [x] **ACONF-03**: Script validates public key derives from private key on configuration. — **Validated**

### Agent Demo - Registration & Auth (AAUTH)

- [x] **AAUTH-01**: User can register an agent with two-step challenge flow (initiate → complete with signature). — **Validated**
- [x] **AAUTH-02**: User can log in an agent using DPoP proof authentication. — **Validated**
- [x] **AAUTH-03**: User can log out an agent by revoking session. — **Validated**
- [x] **AAUTH-04**: User can query current agent info via /agents/me endpoint. — **Validated**
- [x] **AAUTH-05**: User can rotate agent keys with dual-signature verification. — **Validated**

### Agent Demo - Queries (AQUERY)

- [x] **AQUERY-01**: User can query agent's OAuth authorization history. — **Validated**
- [x] **AQUERY-02**: User can query agent's current overseer information. — **Validated**

### Agent Demo - Claim & Oversight (ACLAIM)

- [x] **ACLAIM-01**: User can complete a claim challenge using session or DPoP auth. — **Validated**
- [x] **ACLAIM-02**: User can revoke current overseer relationship. — **Validated**

### Agent Demo - OAuth Client (AOAUTH)

- [x] **AOAUTH-01**: User can register an OAuth client as an agent with key generation or provision options. — **Validated**
- [x] **AOAUTH-02**: User can initiate OAuth authorization for a client, receiving authorization code. — **Validated**

### Client Demo - Configuration (CCONF)

- [x] **CCONF-01**: User can configure client demo via .env file with backend URL, client ID, and Ed25519 keys. — **Validated**
- [x] **CCONF-02**: User can generate PKCE code verifier and challenge pair. — **Validated**
- [x] **CCONF-03**: User can generate Ed25519 keypairs for client authentication. — **Validated**

### Client Demo - Token Flow (CTOKEN)

- [x] **CTOKEN-01**: User can perform token exchange with HTTP callback server and client assertion JWT. — **Validated**
- [x] **CTOKEN-02**: User can refresh access tokens using refresh token grant. — **Validated**
- [x] **CTOKEN-03**: User can revoke access or refresh tokens. — **Validated**

### Client Demo - OAuth Operations (COAUTH)

- [x] **COAUTH-01**: User can query userinfo endpoint with DPoP-bound access token. — **Validated**
- [x] **COAUTH-02**: User can introspect tokens to check status and metadata. — **Validated**
- [x] **COAUTH-03**: User can query OpenID discovery endpoint. — **Validated**
